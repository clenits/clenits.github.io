# BRD: LocalConvert (가칭)

- 버전: v0.1 (Draft)
- 배포 대상: GitHub Pages (정적 사이트)
- 핵심 컨셉: **서버 업로드 없음 / 100% 로컬 처리 / 개인정보 보호**

## 1. 비즈니스 목적

- 사용자가 파일을 서버에 업로드하지 않고 브라우저에서 로컬로 파일 변환을 수행할 수 있는 신뢰형 웹 서비스를 제공한다.
- 초기에는 이미지 변환에 집중하고, 이후 텍스트/오디오/비디오로 확장 가능한 서비스 구조를 마련한다.
- 장기적으로 Google AdSense 기반 수익화가 가능한 콘텐츠/정책 구조를 확보한다.
- 영어/스페인어/중국어/한국어/일본어 사용자에게 동일한 핵심 가치("로컬 처리/프라이버시")를 전달해 글로벌 유입 기반을 만든다.

## 2. 배경 및 문제 정의

- 다수의 온라인 변환 서비스는 파일 업로드를 요구해 개인정보/기밀 유출 우려가 있다.
- 사용자는 설치 없이 빠르게 변환하길 원하지만 로컬 처리 신뢰를 명확히 제공하는 서비스는 제한적이다.
- 정적 사이트 기반으로 운영비를 최소화하면서 초기 트래픽을 확보할 수 있는 모델이 필요하다.

## 3. 릴리즈 범위

### 3.1 In-Scope (v0.1)

- 이미지 변환 중심 정적 웹 서비스
- 브라우저 로컬 처리 기반 변환 플로우
- "서버 업로드 없음" 가치 제안을 전면에 노출하는 UI
- 5개 언어 지원(영어, 스페인어, 중국어, 한국어, 일본어)
- 화면 상단 구석의 국기 마크 기반 언어 선택 UI
- GitHub Pages 배포
- AdSense 승인 대비 기본 페이지(정책/소개/문의) 구성

### 3.2 Out-of-Scope (v0.1)

- 서버/백엔드 개발
- 계정/로그인/저장/히스토리/클라우드 보관
- 텍스트/오디오/비디오 실제 변환 기능 (탭/구조만 To-Be)
- 업로드 기반 AI 기능(배경 제거, 업스케일 등)

## 4. 타겟 사용자

- 회사/학교/개인 작업에서 기밀/개인정보 포함 이미지를 변환해야 하는 사용자
- 설치형 도구 없이 빠른 변환을 원하는 사용자
- 모바일/데스크탑에서 즉시 변환을 원하는 사용자

## 5. 정책/신뢰/수익화 요구사항

### 5.1 프라이버시 원칙

- 파일 데이터는 브라우저 메모리에서만 처리한다.
- 파일명, EXIF 등 민감 정보를 외부 전송하지 않는다.
- 개인정보처리방침에 "서버 업로드 없음" 원칙을 명시한다.

### 5.2 네트워크 허용 범위

- 정적 리소스 로딩(HTML/CSS/JS)
- (추후) AdSense 스크립트 로딩
- (선택) 익명 집계 Analytics

### 5.3 금지 항목

- 파일 내용/파일명/해시/픽셀 데이터 전송
- 업로드 파일 크기 상세 로그 등 식별 가능성이 높은 수집

## 6. SEO 및 AdSense 준비

### 6.1 SEO 최소 요건

- 각 페이지 title/description/meta 설정
- OG 태그 구성
- sitemap.xml, robots.txt 제공
- H1/H2 구조화("로컬 변환" 키워드 포함)
- 다국어 페이지/문구 운영 시 검색엔진이 언어를 인지할 수 있도록 언어별 메타/구조를 관리

### 6.2 AdSense 승인 대비 요건

- 개인정보처리방침 페이지
- 서비스 소개/운영자 정보(최소 이메일)
- 문의 경로(페이지 또는 footer 명시)
- 기능 설명/FAQ 등 기본 콘텐츠 확보
- 광고 영역은 승인 전 비활성 또는 보수적으로 운영

## 7. 단계별 로드맵

- M1: UI 스켈레톤 + To-Be 탭 + GitHub Pages 배포
- M2: 이미지 단일 변환 + 다운로드
- M3: 배치 변환 + ZIP + 옵션(품질/리사이즈/EXIF)
- M4: SEO/정책 페이지 보강 + AdSense 승인 준비

## 8. 향후 확장(To-Be)

- Text: txt/md/docx/pdf 로컬 파싱/변환
- Audio: mp3/wav/aac 변환 (WASM 검토)
- Video: mp4/webm 변환 (ffmpeg.wasm 기반 검토)
- Pro 기능 후보: 대량 처리 UX, 오프라인 PWA, 프리셋 저장(로컬스토리지)

## 9. 다국어 운영 요구사항

- 지원 언어: English, Espanol, 中文, 한국어, 日本語
- 언어 선택 진입점: 화면 상단 구석의 국기 마크 버튼
- 기본 정책: 사용자 직접 선택을 우선하고, 미선택 시 브라우저 언어를 참고해 기본 언어를 노출
- 번역 범위: 핵심 UI 카피, 기능 안내 문구, 정책/FAQ/소개/문의 텍스트
- 원칙: 언어가 달라도 프라이버시 메시지와 핵심 기능 흐름은 동일해야 한다.
