# PRD: LocalConvert (가칭)

- 버전: v0.1 (Draft)
- 배포 대상: GitHub Pages (정적 사이트)
- 핵심 컨셉: **서버 업로드 없음 / 100% 로컬 처리 / 개인정보 보호**

## 1. 제품 목표

- 브라우저에서 파일을 로컬로 변환하는 이미지 변환 MVP를 제공한다.
- 사용자가 "서버 업로드 없음"을 명확히 인지할 수 있는 UX를 제공한다.
- 이후 텍스트/오디오/비디오 확장을 고려해 정보 구조와 UI 탭을 선반영한다.
- 영어/스페인어/중국어/한국어/일본어 5개 언어로 동일 기능을 제공한다.

## 2. 사용자 및 핵심 시나리오

### 2.1 타겟 사용자

- 기밀/개인정보 포함 이미지를 안전하게 변환하려는 사용자
- 설치 없이 빠르게 변환하려는 사용자
- 모바일/데스크탑 환경을 모두 사용하는 사용자

### 2.2 User Stories

1. 이미지를 드래그&드롭해 로컬로 로드하고 JPG로 변환해 다운로드한다.
2. 여러 이미지를 한 번에 넣고 WebP로 일괄 변환해 ZIP으로 받는다.
3. "서버 업로드 없음" 안내를 보고 안심하고 사용한다.
4. 변환 전/후 용량 비교를 보고 압축 강도를 조절한다.

## 3. 기능 요구사항 (이미지 변환 MVP)

### 3.1 입력/출력 포맷

- 입력(최소): PNG, JPG/JPEG, WebP, GIF(정적)
- 출력(최소): PNG, JPG, WebP
- To-Be: AVIF, GIF 애니메이션 처리, HEIC(가능 여부 별도 검증)

### 3.2 변환 기능

- 단일 파일 변환
- 다중 파일 변환(batch)
- 품질 옵션(JPG/WebP quality 1~100)
- 리사이즈 옵션(가로/세로, 비율 유지, 최대 폭/높이 제한)
- 메타데이터 처리
- 기본값: EXIF 제거
- 옵션: EXIF 유지

### 3.3 다운로드 동작

- 단일 변환: 즉시 다운로드
- 다중 변환: ZIP 다운로드
- 파일명 규칙
- 원본이름 + `_converted` + `.ext`
- 중복 시 `(1)`, `(2)` suffix

### 3.4 로컬 처리 보장

- 파일 데이터의 외부 서버 전송 금지
- 네트워크 요청 허용 범위
- 정적 리소스 로딩
- (추후) AdSense 스크립트
- (선택) 익명 집계 Analytics
- UI에 "파일 관련 Network 요청 없음" 안내를 명시한다.

### 3.5 다국어 지원(i18n)

- 지원 언어: English, Espanol, 中文, 한국어, 日本語
- 언어 선택 컴포넌트: 화면 상단 구석의 국기 마크 버튼 + 언어 목록
- 초기 언어 결정: 브라우저 언어 자동 감지 후 지원 언어와 매칭, 미매칭 시 영어 기본값
- 사용자 선택 언어는 재방문 시 유지(LocalStorage)
- 번역 적용 범위: 헤더/버튼/옵션/오류 메시지/정책/FAQ/소개/문의
- 파일 데이터 처리 동작은 언어 변경과 무관하게 동일해야 한다.

## 4. 비기능 요구사항

### 4.1 성능

- 데스크탑: 10MB 이미지 1장 변환 시 목표 1~3초
- 모바일: 저성능/메모리 제한 시 대용량 경고(예: 30MB+)

### 4.2 보안/프라이버시

- 파일은 브라우저 메모리 내 처리
- 파일명/EXIF 민감 정보 전송 금지
- Third-party 스크립트 도입 시 범위와 위험 고지

### 4.3 접근성/호환성

- Chrome/Edge/Safari/Firefox 최신 버전 지원
- 모바일 터치 업로드 지원
- 키보드 접근성 (focus, 명확한 라벨)

## 5. UX/UI 요구사항

### 5.1 상단 고정 메시지

- "파일은 업로드되지 않습니다"
- "100% 브라우저 로컬 처리"
- "개인정보 보호 (EXIF 제거 기본)"

### 5.2 화면 구성

- 헤더: 서비스명 + 탭(Image / Text To-Be / Audio To-Be / Video To-Be) + 상단 구석 국기 마크 언어 선택
- 메인: 드래그&드롭 영역 + 파일 선택 버튼
- 옵션 패널: 출력 포맷/품질/리사이즈/EXIF
- 결과 리스트: 파일명/원본 용량/변환 용량/다운로드
- 배치 액션: 전체 다운로드(ZIP)
- 하단: 소개/FAQ/개인정보처리방침/이용약관/문의

### 5.3 오류/예외 UX

- 미지원 포맷: 지원 포맷 포함 안내 메시지
- 과대 용량/과다 파일: 제한 안내 및 분할 업로드 유도
- 변환 실패: 가능한 원인 + 해결 팁 제공
- 언어 미지원 또는 번역 로드 실패 시: 영어 fallback + 사용자 안내 메시지 제공

## 6. 기술 요구사항

- 정적 사이트 기반(SPA 또는 멀티 페이지)
- 빌드 방향
- v0: 순수 HTML/CSS/JS (빠른 출시)
- v1: Vite/React (확장성)
- 변환 구현 후보
- 네이티브 브라우저 API + Canvas
- WASM 이미지 코덱(확장/성능 필요 시)

## 7. 수용 기준 (Acceptance Criteria)

- PNG/JPG/WebP 입력 → PNG/JPG/WebP 출력 가능
- 단일/다중 파일 변환 및 다운로드 가능
- 다중 변환 ZIP 다운로드 가능
- "서버 업로드 없음" 메시지가 UI에 명확히 노출
- GitHub Pages에서 정상 동작
- 개인정보처리방침/문의/소개 페이지 제공
- 영어/스페인어/중국어/한국어/일본어 전환이 상단 구석 국기 버튼에서 동작
- 선택 언어 상태가 재방문 시 유지
